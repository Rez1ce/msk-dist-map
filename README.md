# msk-dist-map — Проект карты акций по станциям

## Обзор проекта

**msk-dist-map** — это веб-приложение на Go, которое отображает активные акции по станциям метро на интерактивной карте Москвы. Приложение предоставляет REST API для получения и создания акций, а также веб-интерфейс с визуализацией данных.

### Основные возможности

- **REST API**:
  - `GET /api/promos/active` — получение списка активных акций (за последние 6 часов)
  - `POST /api/promos` — создание новой акции
- **Веб-интерфейс**: интерактивная карта (Leaflet) с маркерами станций и таблица со списком акций
- **Автообновление**: данные обновляются каждые 10 секунд
- **Цветовая индикация**: маркеры окрашиваются в зависимости от «свежести» акции:
  - 🟢 зелёный — до 30 минут
  - 🟠 оранжевый — до 120 минут
  - 🔴 красный — старше 120 минут

### Схема базы данных

**Таблица `promos`**:
- `id` — PRIMARY KEY AUTOINCREMENT
- `station_id` — ID станции из `sub_view`
- `created_at` — время создания (автоматически)
- `source_message_id` — заметка (TEXT, опционально)
- `expires_at` — время завершения (NULL = активная)

**Представление `sub_view`** (существует в БД заранее):
- `ID` — ID станции
- `NameOfStation` — название
- `Latitude_WGS84`, `Longitude_WGS84` — координаты

## Сборка и запуск

### Требования

- Go 1.25.5 или новее
- SQLite3 (для драйвера `go-sqlite3`)

### Установка зависимостей

```bash
go mod download
```

### Запуск сервера

```bash
go run .
```

Сервер запустится на порту **8080**:
- Веб-интерфейс: http://localhost:8080
- API: http://localhost:8080/api/promos/active

### Сборка бинарника

```bash
go build -o msk-dist-map.exe
```

### REST API

| Метод | Путь | Описание |
|-------|------|----------|
| `GET` | `/api/promos/active` | Получение списка активных акций (`expires_at IS NULL`) |
| `POST` | `/api/promos/create` | Создание новой акции |
| `PUT` | `/api/promos/{id}` | Обновление акции (установка `expires_at`) |
| `DELETE` | `/api/promos/{id}` | Удаление акции |

#### GET /api/promos/active

Возвращает список активных акций (без `expires_at`).

**Ответ**:
```json
[
  {
    "ID": 1,
    "StationName": "Охотный ряд",
    "Lat": 55.756,
    "Lon": 37.617,
    "CreatedAt": "2026-03-28T10:00:00Z",
    "AgeMinutes": 15
  }
]
```

#### POST /api/promos/create

Создаёт новую акцию. `created_at` устанавливается автоматически (текущее время).

**Тело запроса**:
```json
{
  "station_name": "Охотный ряд",
  "source_message_id": "заметка о промо"
}
```

`source_message_id` — необязательное поле для заметки.

**Ответ** `201 Created`:
```json
{
  "station_id": 42,
  "created_at": "2026-04-05T10:00:00Z"
}
```

#### PUT /api/promos/{id}

Завершает акцию. `expires_at` устанавливается автоматически (текущее время). Тело запроса не требуется.

**Ответ**: `200 OK`

#### DELETE /api/promos/{id}

Удаляет акцию по ID.

**Ответ**: `200 OK`

## Структура кода

| Файл | Описание |
|------|----------|
| `main.go` | Инициализация БД, схема, запуск HTTP-сервера |

| `db.go` | Модели `Station`, `Promo`; функции `getActivePromos()`, `insertPromo()` |
| `static/index.html` | Single-page приложение с картой и таблицей |

## Примечания

- Для SQLCipher (шифрование БД) требуется сборка драйвера с поддержкой шифрования
- Координаты станций берутся из представления `sub_view` (предполагается наличие в БД)
- Станции без координат создаются с пустыми `lat/lon`, которые можно обновить позже
